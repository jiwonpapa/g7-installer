# 보안 정책

## 지원 범위

최신 GitHub Release만 보안 수정 대상입니다. 설치 바이너리는 Release의
`checksums.txt` 검증을 통과한 경우에만 사용하십시오.

## 제보

취약점은 공개 이슈에 비밀값이나 재현 서버 정보를 올리지 말고 GitHub 저장소의
Security 탭에서 비공개 보안 권고로 제보해 주십시오. 영향 범위, 재현 절차,
악용 가능 조건을 포함해 주십시오.

## 운영 원칙

- `g7inst`는 신규 Ubuntu VPS 전용이며 설치·초기화는 root 권한으로 실행됩니다.
- 설치 전 스냅샷을 만들고 `doctor`, `plan`, `reset --dry-run` 결과를 확인하십시오.
- 인증서와 운영 데이터는 초기화 정책에서 별도 보존 대상으로 취급합니다.
